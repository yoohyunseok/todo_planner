## 1. 제품 개요 및 목표

### Executive Summary

Daily Todo MVP는 기본 CRUD, 카테고리 분류, 진행률 추적을 제공하는 단일 HTML 파일 앱이다. 그러나 **마감일 관리, 기한 초과 시각화, 요약 대시보드**가 없어 실무·학습 일정을 통합 관리하기 어렵다. v2.0은 이 공백을 채우면서 단일 HTML 파일 제약을 유지한다.

### 문제 정의

현재 사용자는 할 일을 기록해도 *언제까지 해야 하는지*를 앱 안에서 알 수 없다. 기한이 지난 항목과 오늘 마감인 항목이 완료 전 항목과 동일하게 표시되어 우선순위 판단이 어렵다. 카테고리별 달성 현황도 단일 진행률 바로만 제공되어 업무·개인·공부 영역별 균형을 파악하기 힘들다.

### 제안 솔루션

기존 아키텍처(Single HTML File, localStorage, 순수 JS)를 유지하면서 데이터 모델에 `deadline` 필드를 추가하고, 상단에 요약 대시보드를, 필터 영역에 정렬 컨트롤을, 각 할 일 행에 기한 칩을 삽입한다. 기한 초과·당일 마감 항목은 시각적으로 구분되어 즉시 눈에 띈다.

### 비즈니스 목표

| 목표 | 지표 | 6개월 목표 |
|---|---|---|
| 사용자 재방문율 향상 | 주 3회 이상 사용자 비율 | 현재 대비 +30% |
| 할 일 완료율 향상 | 생성 후 7일 내 완료율 | ≥ 65% |
| 기한 초과 감소 | 기한 경과 미완료 항목 비율 | ≤ 15% |
| 기능 만족도 | 설문 CSAT 점수 (1–5) | ≥ 4.2 |

## 2. 목표 사용자

### Primary Persona — 직장인 김민준 (32세)

**역할:** 중소기업 마케터. 매일 10–20개의 업무 태스크를 관리하며 미팅·보고서·마감이 혼재한다.

**Pain points:** 마감일을 별도 캘린더에 적어야 하고, 오늘 해야 할 일을 한눈에 보기 어렵다. 카테고리 구분 없이 항목이 쌓이면 중요한 것을 놓친다.

**목표:** 앱 하나로 당일 마감·이번주 마감·카테고리별 상태를 즉시 파악하고 싶다.

### Secondary Persona — 취업 준비생 이수진 (25세)

**역할:** 공시 준비생. 공부·개인 일정을 함께 관리하며 모바일과 데스크톱을 번갈아 사용한다.

**Pain points:** 공부 계획 달성률을 모르고, 기한이 지난 과제를 완료 항목과 구분하기 어렵다.

**목표:** 카테고리별 진행률을 시각으로 확인하고, 기한 경과 항목을 즉시 알아채고 싶다.

### 스코프 외 사용자

팀 협업(다중 사용자 동기화)이 필요한 사용자, 서버 연동 백업이 필요한 사용자, 반복(Recurring) 할 일이 주 요구사항인 사용자는 v2.0 대상이 아니다.

## 3. 사용자 스토리

> 우선순위: **MUST** = 이번 버전 필수 / **SHOULD** = 강력 권장 / **COULD** = 여건 허용 시 구현

### US-01 마감일 입력 `MUST`

> 직장인으로서, 할 일을 추가할 때 마감일(날짜)을 선택할 수 있다. 그래야 기한을 별도로 기억하지 않아도 된다.

**인수 기준:**
- 입력 영역에 날짜 선택 필드가 존재하며, 비어 있어도 항목을 추가할 수 있다 (선택 필드).
- 날짜는 YYYY-MM-DD 형식으로 저장된다.
- 수정 모드에서도 마감일을 변경하거나 제거할 수 있다.
- 미래·과거 날짜 모두 입력 가능하다.

### US-02 기한 초과 시각 구분 `MUST`

> 사용자로서, 기한이 지난 미완료 항목이 붉은 색으로 강조되어 눈에 띈다. 그래야 즉시 우선순위를 파악할 수 있다.

**인수 기준:**
- 오늘 날짜 이전 마감이면서 미완료인 항목은 빨간 배경·아이콘으로 구분된다.
- 완료 처리된 기한 초과 항목은 강조를 해제한다.
- 마감일이 없는 항목은 강조하지 않는다.

### US-03 당일 마감 강조 `MUST`

> 사용자로서, 오늘 마감인 항목이 주황 배경으로 표시된다. 그래야 오늘 꼭 완료해야 할 일을 바로 알 수 있다.

**인수 기준:**
- 마감일이 오늘 날짜와 같고 미완료인 항목에 주황 칩이 표시된다.
- 완료 처리 시 강조를 제거한다.

### US-04 요약 대시보드 `MUST`

> 사용자로서, 앱 상단에서 전체·완료·기한 초과·오늘 마감 수와 카테고리별 진행률을 한눈에 본다. 그래야 현재 상태를 빠르게 파악할 수 있다.

**인수 기준:**
- 대시보드에는 총 항목 수, 완료 수, 기한 초과 수, 오늘 마감 수 4개의 통계 카드가 표시된다.
- 카테고리별(업무/개인/공부) 미니 진행률 바가 표시된다.
- 할 일 추가·완료·삭제 시 대시보드가 즉시 갱신된다.

### US-05 마감일·생성일·카테고리 정렬 `MUST`

> 사용자로서, 목록을 마감일 오름차순·생성일 역순·카테고리 기준으로 정렬할 수 있다. 그래야 상황에 따라 원하는 순서로 볼 수 있다.

**인수 기준:**
- 정렬 기준 선택 UI가 필터 영역에 위치한다.
- 마감일 정렬 시 마감일 없는 항목은 맨 뒤로 배치된다.
- 선택한 정렬 기준은 페이지 새로고침 후에도 유지된다 (localStorage).

### US-06 일괄 선택 삭제 `MUST`

> 사용자로서, 여러 항목을 선택해 한 번에 삭제할 수 있다. 그래야 완료된 항목을 빠르게 정리할 수 있다.

**인수 기준:**
- MVP와 동일한 선택 모드(둥근 체크박스, 전체 선택, 선택 삭제)가 유지된다.
- 선택 삭제 전 확인 다이얼로그("N개 항목을 삭제하시겠습니까?")가 나타난다.

### US-07 자동 카테고리 분류 `SHOULD`

> 사용자로서, 할 일 텍스트를 입력하면 키워드 기반으로 카테고리가 자동 선택된다. 그래야 매번 직접 고르지 않아도 된다.

**인수 기준:**
- MVP의 KEYWORD_MAP 로직을 v2.0에서 그대로 유지·확장한다.
- "✦ 자동 분류" 배지가 표시되며 사용자가 직접 변경하면 배지가 사라진다.

### US-08 반응형 모바일 지원 `MUST`

> 모바일 사용자로서, 스마트폰에서도 모든 기능을 불편 없이 사용할 수 있다.

**인수 기준:**
- 560px 이하에서 입력 행이 줄 바꿈, 대시보드는 2열 그리드로 표시된다.
- 액션 버튼이 항상 표시된다 (hover 숨김 해제).
- 터치 영역이 최소 44×44px를 만족한다.

## 4. 확장 데이터 모델

MVP의 Todo 객체에 `deadline` 필드를 추가한다. 기존 필드는 변경하지 않아 하위 호환성을 유지한다.

### Todo 객체 스키마

```js
{
  id        : string,          // crypto.randomUUID() 또는 fallback. 불변.
  text      : string,          // 최대 200자. 공백만인 경우 입력 불가.
  category  : '업무'|'개인'|'공부', // 이외 값은 sanitize 시 '개인'으로 대체.
  completed : boolean,         // 생성 시 false.
  createdAt : string,          // ISO 8601 타임스탬프.
  deadline  : string | null,   // NEW: 'YYYY-MM-DD' 또는 null. 선택 필드.
}
```

### 필드 상세 명세

| 필드 | 타입 | 필수 여부 | 설명 / 제약 |
|---|---|---|---|
| `id` | string | Required | `crypto.randomUUID()` 또는 fallback. 불변. |
| `text` | string | Required | 최대 200자. 공백만인 경우 입력 불가. |
| `category` | string | Required | `'업무' \| '개인' \| '공부'`. 이외의 값은 sanitize 시 `'개인'`으로 대체. |
| `completed` | boolean | Required | 생성 시 `false`. |
| `createdAt` | string | Required | ISO 8601 타임스탬프. 생성 시 `new Date().toISOString()`. |
| `deadline` | string \| null | **Optional (NEW)** | `YYYY-MM-DD` 형식 또는 `null`. 기한 없는 경우 `null`. sanitize 시 형식 불일치는 `null`로 처리. |

### 앱 상태 변수 (추가 / 변경)

| 변수 | 타입 | 변경 여부 | 설명 |
|---|---|---|---|
| `todos` | Array | 유지 | 단일 진실 소스. |
| `editingId` | string\|null | 유지 | 인라인 편집 중인 항목 ID. |
| `activeFilter` | string | 유지 | `'all' \| '업무' \| '개인' \| '공부'` |
| `selectionMode` | boolean | 유지 | 일괄 선택 모드 활성 여부. |
| `selectedIds` | Set\<string\> | 유지 | 선택된 항목 ID 집합. |
| `userOverrideCat` | boolean | 유지 | 사용자가 카테고리 직접 변경 시 true. |
| `activeSort` | string | **신규** | `'createdAt_desc' \| 'deadline_asc' \| 'category_asc'`. localStorage에 저장. |

### localStorage 키

| 키 | 타입 | 설명 |
|---|---|---|
| `'daily-todos'` | JSON 배열 | 기존 키 유지. deadline 필드 포함. |
| `'daily-todos-sort'` | string | **신규.** 마지막 정렬 기준 저장. |

### Sanitize 로직 변경

기존 `sanitize()`에 `deadline` 처리를 추가한다. `/^\d{4}-\d{2}-\d{2}$/` 정규식을 통과하지 못하는 값은 `null`로 대체한다. 기존 MVP 데이터는 `deadline: null`로 마이그레이션된다.

## 5. 기능 명세

### F-01 대시보드 요약 패널 `MUST`

입력 영역 위 또는 헤더 아래에 항상 표시되는 통계 패널. 4개의 통계 카드와 카테고리별 미니 진행률 바로 구성된다.

**기능 요구사항:**

1. 통계 카드: **전체**(todos.length), **완료**(completed 수), **기한 초과**(deadline < today && !completed), **오늘 마감**(deadline === today && !completed) 4개 표시.
2. 카테고리 미니 진행률: 업무·개인·공부 각각 (완료 수 / 전체 수) 비율을 좁은 바 형태로 표시. 항목이 0개인 카테고리는 바를 회색으로 표시.
3. 모든 상태 변화(추가·완료 토글·삭제·편집 저장) 후 즉시 재계산된다.
4. 기한 초과 카드 수가 1 이상이면 카드 텍스트와 숫자를 위험 색(`--danger`)으로 강조한다.
5. 오늘 마감 카드 수가 1 이상이면 경고 색(`--warn`)으로 강조한다.
6. 할 일이 없는 상태에서는 모든 카드 값이 0으로 표시된다.

**비기능 요구사항:**

- NF1. 대시보드 전체 계산은 O(n)으로 단일 패스에서 처리한다.
- NF2. 모바일(≤560px)에서 2×2 그리드로 전환된다.

**엣지 케이스:**

- EC1. 모든 항목이 완료일 때: 완료 카드 = 전체 카드 = N, 기한 초과/오늘 마감 카드 = 0.
- EC2. 자정이 넘어 날짜가 바뀌면 다음 렌더링 시 자동으로 오늘 날짜 기준 재계산된다.

---

### F-02 마감일 필드 (CRUD 통합) `MUST`

할 일 추가 및 수정 폼에 날짜 선택 필드를 추가한다. 마감일은 선택 사항이다.

**기능 요구사항:**

1. 입력 영역(`#input-area`)에 `<input type="date">` 요소 추가. id: `deadline-input`.
2. 비어 있는 상태로 항목 추가 가능 (deadline = null).
3. 추가 후 날짜 필드를 초기화한다.
4. 인라인 수정 모드(`buildEditRow`)에 날짜 필드 추가. 기존 deadline 값을 기본값으로 설정.
5. 수정 저장 시 날짜 필드가 비어 있으면 deadline을 null로 저장.
6. `createTodo(text, category, deadline)` 시그니처로 변경.

**비기능 요구사항:**

- NF1. 날짜 입력 UI는 브라우저 기본 date picker를 사용한다 (외부 라이브러리 없음).
- NF2. iOS Safari, Chrome, Firefox, Edge 최신 버전에서 동작해야 한다.

---

### F-03 기한 상태 시각 강조 `MUST`

각 할 일 행에 마감일 칩을 표시하고, 상태에 따라 배경색과 아이콘을 변경한다.

| 상태 | 조건 | 표시 방식 |
|---|---|---|
| 기한 초과 (Overdue) | deadline < today, !completed | 붉은 배경 (#FEF2F2), "⚠ N일 경과" 칩 |
| 오늘 마감 (Due Today) | deadline === today, !completed | 주황 배경 (#FFFBEB), "오늘 마감" 칩 |
| 다가오는 마감 | deadline > today, !completed | 초록 칩 (배경 변경 없음) |
| 완료 | completed === true | 칩 회색 표시, 행 배경 강조 없음 |

**기능 요구사항:**

1. 마감일 있는 항목의 행에 날짜 칩(`.deadline-chip`)을 표시한다.
2. 기한 초과 칩: "⚠ N일 경과" 텍스트, 붉은 배경(전체 행 배경도 연분홍 적용).
3. 오늘 마감 칩: "오늘 마감" 텍스트, 주황 배경(전체 행 배경도 연주황 적용).
4. 미래 마감 칩: "MM월 DD일" 텍스트, 초록 또는 중립 칩.
5. 완료된 항목의 칩은 회색으로 표시, 전체 행 배경 강조 없음.
6. 날짜 계산은 오늘 날짜를 `new Date().toISOString().slice(0,10)`로 구하여 문자열 비교.

### F-04 정렬 컨트롤 `MUST`

필터 버튼 행 우측에 정렬 드롭다운을 추가한다. 정렬은 필터와 독립적으로 동작한다.

**기능 요구사항:**

1. 정렬 옵션: `createdAt_desc`(최신 순, 기본값), `deadline_asc`(마감일 빠른 순), `category_asc`(카테고리 가나다 순).
2. `deadline_asc` 정렬 시 deadline이 null인 항목은 맨 뒤에 배치.
3. 정렬 후 완료 항목을 맨 뒤로 이동하는 옵션은 v2.0에서 제공하지 않는다 (스코프 외).
4. 선택한 정렬값을 `'daily-todos-sort'` 키로 localStorage에 저장.
5. 정렬 변경 시 현재 선택 모드를 유지하고 selectedIds는 초기화하지 않는다.

**구현 함수:**

`getSorted(list)`: 현재 `activeSort`를 기준으로 배열을 정렬하여 반환. 원본 `todos` 배열을 직접 변경하지 않는다.

렌더링 파이프라인: `getVisible()` → `getSorted(visible)` → `renderTodos()`

---

### F-05 카테고리 필터 (기존 유지·확장) `MUST`

MVP의 카테고리 필터 4개(전체/업무/개인/공부) 버튼을 그대로 유지한다. v2.0에서 필터 영역에 정렬 컨트롤이 추가되므로 레이아웃을 `justify-content: space-between`으로 조정한다.

---

### F-06 전체 완료 배너 (기존 유지) `MUST`

MVP의 "모든 할 일을 완료했습니다!" 배너를 그대로 유지한다.

---

### F-07 자동 카테고리 분류 (기존 유지·확장) `SHOULD`

MVP의 `detectCategory()`, `KEYWORD_MAP`, "✦ 자동 분류" 배지, `userOverrideCat` 로직을 v2.0에서 그대로 유지한다. 키워드 확장은 선택 사항이다.

---

### F-08 진행률 표시 (기존 확장) `MUST`

헤더의 전체 진행률 바는 유지한다. 대시보드(F-01)의 카테고리별 미니 진행률 바가 이를 보완한다.

**기능 요구사항:**

1. 헤더 진행률 바: 전체 todos 기준, 완료율 0–100% 표시.
2. 100% 완료 시 초록색(`--success`)으로 전환.
3. 대시보드 카테고리 진행률 바: 각 카테고리별 독립 계산.

## 12. 미결 질문

**Q1. 날짜 타임존 처리 방식**
`new Date().toISOString().slice(0,10)`은 UTC 기준이다. 한국(UTC+9) 사용자는 자정 전후 1시간 동안 날짜가 어긋날 수 있다. 로컬 날짜 문자열을 직접 구성하는 헬퍼 함수를 사용할지 결정 필요.

**Q2. 대시보드 접힘 여부**
할 일이 없거나 모바일에서 공간이 부족할 때 대시보드를 접을 수 있는 토글 버튼을 제공할지 결정 필요.

**Q3. 마감일 정렬 시 완료 항목 처리**
완료 항목을 정렬 그룹 내에서 미완료 항목 다음에 배치할지, 또는 그룹화 없이 날짜 순으로만 처리할지 결정 필요.

**Q4. 기한 초과 행 강조 색상 충돌**
MVP의 category별 왼쪽 border accent와 기한 초과 배경색이 시각적으로 충돌할 수 있다. 기한 초과 시 border-left 색을 danger로 덮어씌울지 유지할지 결정 필요.

**Q5. 앱 타이틀 변경 여부**
"Daily Todo"에서 "Daily Planner" 또는 "플래너"로 변경할지 결정 필요.

### F-09-1 기능 개요 (Feature Overview)

월간 달력 그리드를 통해 마감일이 있는 할 일들을 한눈에 조회하는 뷰를 제공한다. 사용자는 목록 뷰(List View)와 달력 뷰(Calendar View) 사이를 자유롭게 전환할 수 있다. **뷰 전환은 기존 목록 상태(todos 배열, activeFilter, selectedIds, activeSort)를 일절 변경하지 않는다.** 달력은 별도의 저장 공간이 아니며, todos 배열의 deadline 필드를 읽어 렌더링하는 시각화 레이어다.

**핵심 설계 원칙:**

- 달력 뷰와 목록 뷰는 동일한 todos 배열을 공유한다. 단일 진실 소스(Single Source of Truth) 원칙을 유지한다.
- 달력 뷰에서 날짜를 클릭하면 목록 뷰에 해당 날짜 필터가 적용된다. 달력과 목록은 같은 화면에 공존할 수 있다.
- 달력 뷰는 deadline 필드가 있는 항목만 표시한다. deadline이 null인 항목은 달력에 나타나지 않는다.

### F-09-2 사용자 스토리 (User Stories)

> 우선순위: **SHOULD** — v2.1 강력 권장 기능. deadline 필드(F-02) 구현이 선행되어야 한다.

#### US-CAL-01 월간 달력 조회 `SHOULD`

> 사용자로서, 이번 달의 마감일이 있는 할 일들을 달력에서 한눈에 보고 싶다. 그래야 이번 달 일정을 시각적으로 파악하고 바쁜 날짜를 미리 인지할 수 있다.

**인수 기준:**
- 달력 뷰로 전환하면 현재 월의 월간 그리드가 표시된다.
- deadline이 해당 날짜와 일치하는 할 일이 각 날짜 셀에 칩(chip) 형태로 표시된다.
- deadline이 없는 할 일은 달력에 표시되지 않는다.
- 달력 뷰 전환 시 기존 목록의 항목, 필터, 정렬 상태가 변경되지 않는다.

#### US-CAL-02 날짜 클릭으로 할 일 확인 `SHOULD`

> 사용자로서, 날짜를 클릭해서 해당 날의 할 일들을 목록에서 확인하고 싶다. 그래야 특정 날의 마감 항목을 상세히 검토하고 완료 처리할 수 있다.

**인수 기준:**
- 달력 셀을 클릭하면 selectedCalDate가 해당 날짜로 설정되고, 목록 영역이 해당 날짜의 deadline 항목만 표시하도록 필터된다.
- 이미 선택된 날짜 셀을 다시 클릭하면 selectedCalDate가 null로 초기화되어 전체 목록이 복원된다.
- 해당 날짜에 할 일이 없는 빈 날짜 셀도 클릭 가능하다. 클릭 시 "해당 날짜에 마감인 할 일이 없습니다." 빈 상태 메시지가 표시된다.
- 날짜 클릭 필터는 기존 카테고리 필터(activeFilter)와 독립적으로 동작한다. 두 필터는 AND 조건으로 결합된다.

#### US-CAL-03 이전/다음 달 이동 `SHOULD`

> 사용자로서, 달력에서 이전/다음 달로 이동하고 싶다. 그래야 다음 달 마감 일정을 미리 확인하거나 지난 달 완료 현황을 돌아볼 수 있다.

**인수 기준:**
- "◀ 이전" 버튼 클릭 시 calendarMonth가 1 감소하고, 1월에서 클릭하면 calendarYear가 1 감소하고 calendarMonth가 12로 설정된다.
- "다음 ▶" 버튼 클릭 시 calendarMonth가 1 증가하고, 12월에서 클릭하면 calendarYear가 1 증가하고 calendarMonth가 1로 설정된다.
- "오늘" 버튼 클릭 시 calendarYear와 calendarMonth가 현재 연월로 리셋된다.
- 월 이동 시 selectedCalDate는 null로 초기화된다.
- 네비게이션 영역에 현재 연월이 "YYYY년 MM월" 형식으로 표시된다.

#### US-CAL-04 오늘 날짜 강조 `SHOULD`

> 사용자로서, 오늘 날짜가 달력에서 강조 표시되기를 원한다. 그래야 현재 날짜를 즉시 인식하고 오늘 마감인 항목을 바로 확인할 수 있다.

**인수 기준:**
- 오늘 날짜 셀은 날짜 숫자에 --primary 색상의 원형 배경이 적용된다.
- 오늘 날짜 강조는 현재 표시 중인 월에서만 나타난다. 다른 월로 이동하면 강조가 없다.
- 오늘 날짜 셀이 선택(clicked) 상태가 되어도 강조 스타일이 유지된다 (선택 스타일과 오늘 스타일이 공존).

### F-09-3 기능 요구사항 (Functional Requirements)

#### FR-CAL-01 월간 그리드 구조

1. 7열(일/월/화/수/목/금/토) × 5~6행 구조의 CSS Grid로 달력을 렌더링한다.
2. 첫 번째 행은 요일 헤더(일 월 화 수 목 금 토)로 구성된다. 헤더는 스크롤되지 않는다.
3. 각 날짜 셀은 해당 월의 날짜 숫자와 할 일 칩(chip)으로 구성된다.
4. 해당 월의 첫째 날 요일을 기준으로 그 이전 날짜 셀은 직전 월의 날짜를 그레이 처리하여 채운다. 해당 월의 마지막 날 이후 셀은 다음 월의 날짜를 그레이 처리하여 채운다 (현재 월 외 날짜).
5. 28일 달(2월 평년), 29일 달(2월 윤년), 30일 달, 31일 달 모두 정확히 렌더링되어야 한다.

#### FR-CAL-02 날짜 셀 내용

1. **날짜 숫자:** 셀 상단에 표시. 오늘 날짜는 --primary 색 원형 배경으로 강조.
2. **할 일 칩(task chip):** 해당 날짜에 deadline이 있는 할 일을 칩으로 표시. 최대 3개까지 표시. 4개 이상이면 "+N개 더" 텍스트로 초과분을 표시.
3. **칩 스타일:** 칩 텍스트는 8자로 잘라서 표시(말줄임 처리). 칩 배경색은 카테고리 색상에 대응한다 (업무: 파랑 계열, 개인: 초록 계열, 공부: 보라 계열).
4. **기한 초과 인디케이터:** 기한이 지난(deadline < today) 미완료 할 일이 있는 날짜 셀에는 셀 우상단 또는 날짜 숫자 인근에 소형 빨간 점(dot) 또는 연분홍 배경을 적용한다.
5. **현재 월 외 날짜:** 직전/다음 월의 날짜 숫자는 --text-muted 색상으로 흐리게 표시하고, 해당 셀에는 칩을 표시하지 않는다.
6. **선택된 날짜:** selectedCalDate와 일치하는 셀은 --primary-pale 배경과 --primary 테두리로 강조한다.

#### FR-CAL-03 네비게이션

1. 달력 상단에 네비게이션 바를 배치한다. 구성: `◀ 이전` 버튼 — `YYYY년 MM월` 타이틀 — `오늘` 버튼 — `다음 ▶` 버튼.
2. 이전/다음 버튼은 calendarMonth를 1씩 증감하며 연도 경계에서 자동으로 연도를 변경한다 (12월 → 다음 ▶ → 1월, 연도+1).
3. 오늘 버튼은 calendarYear, calendarMonth를 현재 연월로 리셋하고 selectedCalDate를 null로 초기화한다.
4. 월 이동 시 selectedCalDate는 null로 초기화된다.

#### FR-CAL-04 날짜 클릭 필터

1. 셀 클릭 시 selectedCalDate를 "YYYY-MM-DD" 형식으로 설정하고 목록 영역을 해당 날짜의 deadline 항목만 표시하도록 필터한다.
2. 동일 셀 재클릭 시 selectedCalDate를 null로 초기화하고 목록을 전체 복원한다.
3. 날짜 클릭 필터는 카테고리 필터(activeFilter)와 AND 조건으로 결합된다.
   - 예: activeFilter='업무', selectedCalDate='2026-07-15'이면 카테고리가 업무이면서 deadline이 2026-07-15인 항목만 표시된다.
4. 선택된 날짜 필터가 활성화된 상태에서 빈 상태 메시지는 "2026년 7월 15일 마감인 할 일이 없습니다."처럼 날짜를 명시한다.

#### FR-CAL-05 뷰 전환 토글

1. 필터 & 정렬 바(Z4 영역)에 뷰 전환 pill 버튼을 추가한다: `목록` / `달력` 두 개의 pill 버튼.
2. `목록` 버튼 클릭 시 activeView를 'list'로 설정한다. 달력 영역을 숨기고 기존 todo-list 영역을 표시한다.
3. `달력` 버튼 클릭 시 activeView를 'calendar'로 설정한다. todo-list 영역 위 또는 대체 위치에 달력 그리드를 렌더링한다. 목록은 달력 아래에 계속 표시된다 (달력이 목록을 완전히 대체하지 않는다).
4. activeView 값은 localStorage에 저장하지 않는다 (세션 중 상태로만 유지).
5. 뷰 전환 시 todos 배열, activeFilter, activeSort, selectedIds는 변경되지 않는다.

### F-09-4 데이터 요구사항 (Data Requirements)

#### 신규 앱 상태 변수

| 변수 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `activeView` | `'list' \| 'calendar'` | `'list'` | 현재 활성 뷰. 세션 중 유지. localStorage 미저장. |
| `calendarYear` | `number` | 현재 연도 | 달력에 표시 중인 연도. |
| `calendarMonth` | `number` | 현재 월 (1–12) | 달력에 표시 중인 월. 1=1월, 12=12월. |
| `selectedCalDate` | `string \| null` | `null` | 클릭으로 선택된 날짜 ("YYYY-MM-DD"). null이면 날짜 필터 미적용. |

#### 기존 데이터 모델 변경 없음

- Todo 객체에 새 필드를 추가하지 않는다. 달력 뷰는 기존 `deadline` 필드(v2.0에서 추가됨)를 읽기 전용으로 사용한다.
- localStorage 키 변경 없음. `'daily-todos'`와 `'daily-todos-sort'` 키는 그대로 유지된다.

#### 달력 렌더링 데이터 흐름

```
todos (Array)
  └─ deadline 필드가 있는 항목만 필터
      └─ calendarYear + calendarMonth 기준으로 해당 월 그리드 생성
          └─ 각 날짜 셀에 deadline 일치 항목 매핑
              └─ 셀 렌더링 (칩, 오늘 강조, 기한 초과 dot)
```

#### selectedCalDate와 activeFilter의 결합 로직

```js
// getCalendarFiltered(): 달력 날짜 필터 + 카테고리 필터 AND 조건
function getCalendarFiltered() {
  return todos.filter(function(t) {
    var catMatch  = activeFilter === 'all' || t.category === activeFilter;
    var dateMatch = selectedCalDate === null || t.deadline === selectedCalDate;
    return catMatch && dateMatch;
  });
}
```

달력 뷰가 활성화된 상태에서 목록 렌더링은 `getVisible()` 대신 `getCalendarFiltered()`를 사용한다.

### F-09-7 기술 제약 및 구현 가이드라인

#### 단일 파일 제약 준수

- 달력 렌더링 로직은 `renderCalendar()` 함수로 캡슐화하여 기존 `renderTodos()`와 분리한다.
- 달력 뷰 HTML 컨테이너(`#calendar-view`)는 `#list-area` 직전에 삽입한다. 뷰 전환 시 display:none/block으로 제어한다.
- CSS는 기존 디자인 토큰(`--primary`, `--border`, `--surface` 등)을 최대한 재사용한다. 신규 토큰은 필요 최소한으로 추가한다.

#### 날짜 계산

```js
// 달력 그리드 생성 헬퍼 — 해당 월의 1일과 말일을 구함
function getMonthRange(year, month) {
  var firstDay = new Date(year, month - 1, 1);  // month는 1-based
  var lastDay  = new Date(year, month, 0);
  return {
    firstDayOfWeek: firstDay.getDay(),  // 0=일, 6=토
    daysInMonth:    lastDay.getDate(),
    daysInPrevMonth: new Date(year, month - 1, 0).getDate()
  };
}
```

- 모든 날짜 비교는 `YYYY-MM-DD` 문자열 비교로 처리한다. Date 객체를 직접 비교하지 않는다.
- 타임존 이슈 방지를 위해 `getTodayStr()` 헬퍼(v2.0에서 이미 정의됨)를 재사용한다.

#### 렌더링 파이프라인 변경

```
기존: getVisible() → getSorted() → renderTodos()
변경: getCalendarFiltered() → getSorted() → renderTodos()  (달력 뷰에서 selectedCalDate 있을 때)
     renderCalendar()  (달력 뷰 활성 시 별도 호출)
```

모든 상태 변경(추가·삭제·완료 토글·편집 저장) 후 renderTodos()와 함께 renderCalendar()도 호출하여 달력의 칩 상태를 최신으로 유지한다.

#### 성능

- `renderCalendar()`는 todos 배열에서 deadline 있는 항목만 필터한 후, Map 자료구조로 날짜 → 항목 배열 인덱스를 구성하여 O(n) 매핑을 보장한다.
- 셀 개수는 최대 42개(6주 × 7일)이므로 DOM 조작 부하는 무시할 수준이다.
