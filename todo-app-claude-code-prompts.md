# 할 일 관리 앱 — 클로드 코드 단계별 프롬프트

PRD를 클로드 코드(Claude Code)에서 바로 쓸 수 있는 **5개 핵심 단계 프롬프트**로 변환한 문서입니다.
각 단계의 회색 박스 안 내용을 그대로 복사해 순서대로 입력하세요. 이전 단계 결과 위에 점진적으로 쌓이는 구조라 **순서를 지켜 진행**하는 것이 중요합니다.

---

## 단계 1 — 프로젝트 뼈대 & 데이터 저장 기반

```
순수 HTML/CSS/JavaScript만 사용하는 개인용 할 일 관리 웹앱을 만들 거야.
외부 프레임워크나 라이브러리는 절대 사용하지 마.

이번 단계에서는 기본 뼈대와 데이터 저장 기반만 구현해줘:

1. index.html 단일 파일로 시작해. (HTML, CSS, JS를 한 파일에 작성)
2. 화면 레이아웃 영역을 위에서 아래 순서로 자리만 잡아줘:
   - 헤더(앱 제목 "Daily Todo" + 진행률 표시 자리)
   - 입력 영역(텍스트 입력창 + 카테고리 선택 + 추가 버튼)
   - 목록 영역(빈 상태)
3. 할 일 데이터 모델을 다음 구조로 정의해:
   { id, text, category, completed, createdAt }
   - category 값은 "업무" | "개인" | "공부" 중 하나
4. localStorage 읽기/쓰기 헬퍼 함수를 만들어:
   - loadTodos(): localStorage에서 todos 배열을 읽어옴 (없으면 빈 배열)
   - saveTodos(todos): todos 배열을 JSON 문자열로 저장
5. 앱 진입 시 loadTodos()로 데이터를 불러오고, 화면을 렌더링하는
   renderTodos() 함수의 골격을 만들어줘 (지금은 비어 있어도 됨).

CSS는 기본적인 정렬과 가독성만 적용하고, 디자인 완성도는 마지막 단계에서 다룰 거야.
완료 후 어떤 함수와 구조를 만들었는지 간단히 설명해줘.
```

**완료 확인:** `index.html` 하나가 생성되고, 브라우저로 열면 레이아웃 영역이 보이며, 콘솔 에러가 없어야 한다.

---

## 단계 2 — 할 일 추가 & 목록 표시

```
이전 단계에서 만든 index.html에 "추가"와 "표시" 기능을 구현해줘.

1. 할 일 추가:
   - 입력창에 텍스트를 입력하고 추가 버튼 클릭 또는 Enter 키로 등록.
   - 추가 시 선택된 카테고리(업무/개인/공부)를 함께 저장. 미선택 시 기본값 "개인".
   - 공백만 입력된 경우 추가하지 말고 사용자에게 간단히 안내.
   - 새 할 일은 completed: false 상태, 고유 id(생성 시각 기반), createdAt 포함.
   - 추가 직후 입력창을 비워.

2. 목록 표시 (renderTodos 완성):
   - todos 배열을 순회하며 각 항목을 목록 영역에 렌더링.
   - 각 항목은 [완료 체크박스 자리] [할 일 내용] [카테고리 라벨] 을 표시.
     (체크박스/수정/삭제 동작은 다음 단계에서 연결, 지금은 표시까지만)
   - 할 일이 0개면 "할 일이 없습니다" 같은 빈 상태 메시지 표시.

3. 추가할 때마다 saveTodos()로 저장하고 renderTodos()로 화면을 갱신해.

새로고침 후에도 추가한 할 일이 그대로 보이는지 반드시 확인하고 알려줘.
```

**완료 확인:** 할 일을 추가하면 목록에 즉시 나타나고, 새로고침해도 유지된다.

---

## 단계 3 — 완료 체크 · 수정 · 삭제

```
이전 단계의 목록 항목에 완료 체크, 수정, 삭제 기능을 연결해줘.

1. 완료 체크:
   - 각 항목의 체크박스 클릭 시 해당 할 일의 completed 값을 토글.
   - 완료된 항목은 시각적으로 구분(취소선 + 흐린 색상 등).
   - 변경 즉시 saveTodos() 저장 및 화면 갱신.

2. 수정:
   - 각 항목에 수정 버튼을 추가.
   - 수정 모드에서 할 일 내용과 카테고리를 변경할 수 있어야 함.
   - 저장 시 반영, 취소 시 기존 내용 유지.
   - 빈 내용으로 저장은 막아줘.

3. 삭제:
   - 각 항목에 삭제 버튼을 추가.
   - 실수 방지를 위해 삭제 전 confirm 확인 절차를 둬.
   - 삭제 후 saveTodos() 저장 및 화면 갱신.

모든 변경(체크/수정/삭제)이 localStorage에 즉시 반영되고
새로고침 후에도 유지되는지 확인해줘.
```

**완료 확인:** 체크·수정·삭제가 모두 동작하고, 모든 변경이 새로고침 후에도 유지된다.

---

## 단계 4 — 카테고리 구분 · 필터링 · 진행률

```
카테고리 시각화, 필터링, 진행률 기능을 구현해줘.

1. 카테고리 시각 구분:
   - 업무 / 개인 / 공부를 각각 다른 색상의 라벨로 표시.

2. 카테고리 필터:
   - 목록 위에 [전체] [업무] [개인] [공부] 필터 버튼을 추가.
   - 선택된 필터에 해당하는 할 일만 목록에 표시.
   - 현재 선택된 필터를 시각적으로 표시.

3. 진행률 표시 (헤더 영역):
   - 진행률(%) = (완료된 할 일 수 / 전체 할 일 수) × 100, 정수로 표시.
   - 퍼센트 숫자와 함께 막대형 프로그레스 바로 시각화.
   - 할 일이 0개면 0% 또는 "할 일 없음" 처리.
   - 추가/완료/삭제 등 변경 시 실시간으로 갱신.
   - 진행률은 필터와 무관하게 "전체" 기준으로 계산해.

기능 추가 후 진행률 계산이 정확한지 몇 가지 경우로 확인해줘.
```

**완료 확인:** 카테고리가 색으로 구분되고, 필터가 동작하며, 진행률 바가 실시간으로 정확히 갱신된다.

---

## 단계 5 — 디자인 마무리 · 반응형 · 예외 처리 · QA

```
마지막으로 완성도를 높여줘.

1. 디자인 정리:
   - 일관된 색상/여백/타이포로 깔끔하게 정돈.
   - 10~20개 항목을 보기 편한 밀도로 배치.
   - 버튼/입력창에 hover, focus 등 기본 상호작용 스타일 적용.

2. 반응형:
   - 데스크톱과 모바일 브라우저 모두에서 자연스럽게 보이도록 조정.

3. 예외 처리 점검:
   - localStorage 데이터가 손상됐을 때 앱이 깨지지 않도록 안전하게 처리.
   - 빈 목록, 모든 항목 완료(100%) 등 경계 상황 확인.

4. 최종 QA — 아래 완료 기준을 모두 점검하고 결과를 보고해줘:
   - [ ] 할 일 추가·수정·삭제 동작
   - [ ] 완료 체크 토글 및 진행률 반영
   - [ ] 업무/개인/공부 카테고리 분류 및 필터
   - [ ] 진행률(%)·프로그레스 바 실시간 갱신
   - [ ] 새로고침/재실행 후 데이터 유지
   - [ ] 외부 프레임워크 없이 순수 자바스크립트로 구현

남은 이슈가 있으면 정리해서 알려줘.
```

**완료 확인:** PRD의 모든 완료 기준이 충족되고, 데스크톱·모바일에서 안정적으로 동작한다.
